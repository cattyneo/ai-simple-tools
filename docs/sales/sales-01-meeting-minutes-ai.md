# AI商談議事録・ネクストアクション自動生成ツール

## 概要
オンライン商談の音声自動文字起こしから、AIが議事録・ネクストアクション・顧客のニーズを自動抽出し、SFAに即座に連携。商談後の事務作業を最大90%削減。

## 目的・解決する課題
- 営業担当が商談後に手作業で議事録を作成する時間を削減
- ネクストアクション（フォロー内容・期限）の抜け漏れをなくす
- チーム全体で同じ顧客情報をリアルタイムで共有できていない

## ターゲット
営業組織10名以上の中堅企業。特にZoom・Google Meetなどのオンライン商談が増えた営業部門。既存SFA（Salesforce・Pipedrive等）を使用している層。

## 主要機能
- Zoom・Google Meet・Teams との自動連携（ワンクリック録音スタート）
- AI音声認識＋自然言語処理で商談内容を自動要約
- 顧客のニーズ・課題・予算フェーズを自動抽出
- ネクストアクション（営業側の行動）を自動生成＋期限提案
- 議事録をSFA（Salesforce等）に自動入力
- チーム内でテンプレート・ベストプラクティスを共有可能

## 競合との差別化
Fireflies.io等の文字起こしツールは存在するが、議事録生成＋SFA自動入力＋ネクストアクション提案まで一体化したツールは少ない。Salesforce独自の AI Einsteinは高額（年数百万）であり、中堅企業向け廉価版の空白市場。

## 想定価格帯
初期費用: 20万円 / 月額: 3～5万円（ユーザー数に応じた従量課金）

## 開発難易度
中 / Zoom・Google Meet API とのネイティブ連携、および Speech-to-Text（Google Cloud Speech-to-Text、Azure Speech等）とLLM（GPT-4等）の組み合わせで実装可能。SFA連携API設定が手間。

## Agent所見
- **ディレクター**: 営業部門の意思決定スピードが上がり、失注防止に直結。初期導入は丁寧なオンボーディングが必須。
- **エンジニア**: Speech-to-Text + LLM の API 組み合わせは技術的難易度は中程度。SFA毎のAPI仕様差をどう吸収するかが開発コストのポイント。
- **マーケター**:「商談後の事務作業 90% 削減」は営業マネージャーの刺さるメッセージ。デモ動画による視覚的訴求が効果的。
- **セールス**: 既存SFA導入企業がターゲットなので、SFA選定後のスピーディな連携デモが受注のカギ。ユーザー数課金は導入障壁になるため、セット価格の提案が効果的。

## 市場リサーチ補足（market-research準拠）

### 市場規模推定
- **TAM（Total Addressable Market）**: SFA導入企業（日本国内約5,000～10,000社）× 月額3～5万円の市場 ≈ 約180～600億円
- **SAM（Serviceable Addressable Market）**: 営業組織10名以上でZoom/Teams導入済みのSFA企業 ≈ 約2,000～3,000社 ≈ 約72～180億円
- **SOM（Serviceable Obtainable Market）**: 初期3年で狙える市場 ≈ 200～500社 × 年単価50万円 ≈ 10～25億円

### 競合実態
- **Fireflies.io**: 音声起こし＋基本要約のみ。月額10～25ドル（廉価）だが、SFA自動入力なし
- **Otter.ai**: 高精度音声認識が強み。ただし議事録生成・ネクストアクション提案機能は弱い
- **Salesforce Einstein**: ネイティブSFA機能として年数百万円。中小企業には導入障壁が高い
- **本ツールの差別化**: ネクストアクション自動生成 + SFA自動入力が唯一の統合機能。廉価性（Fireflies.ioの3倍程度だが、Einsteinの1/100以下）

### 反証・リスク要因（このツールが売れない可能性）
- **音声認識精度の問題**: 背景ノイズが多い営業現場での認識精度が70%未満に低下する可能性
- **プライバシー・セキュリティ懸念**: 音声録音・自動転写に対する顧客の法的・倫理的懸念（特に医療・金融業界）
- **既存オンボーディング負荷**: SFA多様性（Salesforce/Pipedrive/kintone等）に対応する開発コストが予想以上に高い
- **導入後の利用率低迷**: 商談後に「自動生成議事録」が精度不十分で、結局営業が手修正するケースが30～40%発生
- **ネクストアクション提案精度**: 「次のステップは電話」と提案しても、営業の判断で異なるアクションを取る場合が多く、ユーザー満足度低下の可能性

## 戦略評価（strategy-advisor準拠）

### 開発優先度
- **総合判定: 高**
- **理由**: 営業部門の「ペインの大きさ」（商談後の事務作業25%削減）が明確。ROIメッセージが営業マネージャーに刺さりやすく、初期採用企業から紹介による新規顧客獲得の可能性が高い。

### 想定ROI（初期投資回収期間）
- **初期投資**: 開発費500万 + 営業・マーケティング初期費用200万 = 700万円
- **月額平均客単価**: 4万円
- **想定初期採用速度**: 6か月で20社、12か月で50社
- **初期12か月MRR**: 20社×4万円=80万円 → 50社×4万円=200万円
- **回収期間**: 初期12か月で約200万円×12か月 = 2,400万円 → 3年以内に回収可能
- **ROI指標**: 初期投資700万 / 3年LTV（50社×4万円×36か月）= 6,000万 → **ROI: 8.6倍（3年ベース）**

### 成功指標（KPI）
- **短期（6か月）**: ベータテスト企業10社からの定性フィードバック「業務時間削減50%以上」達成
- **中期（12か月）**:
  - 有料顧客: 50社
  - MRR: 200万円
  - NPS（Net Promoter Score）: 55以上
  - 顧客の「議事録作成時間削減率」: 平均60%以上
- **長期（24か月）**:
  - 有料顧客: 150社
  - MRR: 600万円
  - ARR成長率: 月間10%以上
  - 顧客紹介による新規リード: 全新規リードの35%以上

### コンティンジェンシープラン（リスク対応）
**リスク1: 音声認識精度が期待値を下回る場合**
- 対応: 複数音声API（Google Cloud Speech + Azure + Anthropic）を並列利用し、精度向上を図る。ユーザーによる手動修正UI充実化。

**リスク2: SFA連携の実装が予想以上に複雑化する場合**
- 対応: 初期段階ではCSV出力機能のみに限定し、フェーズ2以降にSFA API連携を段階展開。

**リスク3: 導入後の継続率が60%以下（離脱率40%以上）の場合**
- 対応: 初期3か月のカスタマーサクセス支援を無料で実施。定期的なユーザーセミナー開催。
