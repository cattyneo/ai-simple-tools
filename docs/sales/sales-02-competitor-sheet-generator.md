# 競合比較資料自動生成ツール

## 概要
競合企業のHP・料金ページ・プレスリリース等の公開情報をAIが自動収集・分析し、自社との比較表を数分で自動生成。営業が手作業で競合調査に費やす時間を大幅削減。

## 目的・解決する課題
- 営業が見込み客との商談で「他社との違いは？」という質問に十分な回答ができていない
- 競合分析資料の作成・更新に営業が多くの時間を割いている
- 競合情報が古いままでアップデートされていない

## ターゲット
BtoB営業組織（特にSaaS企業・コンサルティングファーム）。営業組織20名以上で、営業資料の質・スピード競争が激しい組織。

## 主要機能
- 競合企業URLの一括登録（複数URL同時対応）
- Web スクレイピング + AI による競合企業の事業内容・機能・料金の自動抽出
- 自社製品情報との自動比較表生成（Excel / Google Sheets / PowerPoint対応）
- 競合企業の新着プレスリリース・情報更新の自動検知
- 差別化ポイント（自社の優位性）を AI が自動提示
- 比較資料の自動アップデート機能（月1回等、定期実行可能）

## 競合との差別化
Semrush や SimilarWeb は SEO・トラフィック分析ツール。本ツールは「営業資料化」に特化しており、自社との比較表をワンクリックで生成できる。Gartner Magic Quadrant は高額で、リアルタイム更新は不可。中小企業の営業部門向けの廉価競合分析ツールは未成熟。

## 想定価格帯
初期費用: 30万円 / 月額: 2～4万円（競合企業登録数に応じた従量課金）

## 開発難易度
中 / Web スクレイピング（Beautiful Soup 等）+ LLM（GPT-4 Vision）を組み合わせることで実装可能。法的リスク（スクレイピング規約）の確認が必須。

## Agent所見
- **ディレクター**: 営業資料の質がAIで一定化するメリットは大。ただしスクレイピングの法的リスク（利用規約等）の早期確認が重要。
- **エンジニア**: Webスクレイピング + LLM の組み合わせは実装可能。競合サイトの構造変更への対応（監視・自動修正）がメンテナンスコストになる点は要注意。
- **マーケター**: 「営業資料化」という軸で訴求することで、従来の競合分析ツールとの差別化が明確。営業部門向けのデモが効果的。
- **セールス**: SaaS・コンサルティング企業は競合との差別化に敏感で、ウィルスループが期待できる市場。初期導入時に「競合10社の比較資料」テンプレートを無料作成してあげることで導入促進。

## 市場リサーチ補足（market-research準拠）

### 市場規模推定
- **TAM（Total Addressable Market）**: 営業資料作成ニーズのある企業（日本国内約8,000～12,000社） × 月額2～4万円 ≈ 約200～600億円
- **SAM（Serviceable Addressable Market）**: SaaS・コンサルティング企業で営業組織20名以上 ≈ 約1,500～2,500社 ≈ 約36～120億円
- **SOM（Serviceable Obtainable Market）**: 初期3年で狙える市場 ≈ 150～300社 × 年単価36万円 ≈ 5.4～10.8億円

### 競合実態
- **Semrush**: SEO・トラフィック分析中心。競合比較資料機能は付随的。月額119～449ドル（高額）
- **SimilarWeb**: Webトラフィック分析特化。営業資料化機能なし
- **Competitor.com**: 営業資料化を標榜するが、自動生成度が低く、手修正が多い
- **本ツールの差別化**: 「営業資料化特化」「自動比較表生成」が唯一の統合機能。廉価性で優位。

### 反証・リスク要因（このツールが売れない可能性）
- **Web スクレイピング法的リスク**: 競合サイトのスクレイピングが利用規約で禁止されている場合、法的紛争リスク高
- **データ鮮度の問題**: 競合企業の料金・機能は頻繁に変更され、スクレイピングベースでは常に更新遅延が発生
- **精度不足**: LLMによる比較分析に誤りが発生し、営業資料として使用不可なケースが30%以上発生する可能性
- **競合企業からの苦情**: 著作権侵害や不正確な情報記載で、競合企業から法的請求を受けるリスク
- **営業のニーズズレ**: 実際には営業は「自社とライバル企業の差別化ポイント」だけを欲しており、完全な比較表は不要な場合が多い

### データソース
- Semrush、SimilarWeb公式プライシングページ
- Competitor.com ユーザーレビュー（G2, Capterra）
- Gartner Magic Quadrant: Competitive Intelligence Platforms（2024年版）
- 日本弁護士会「Web スクレイピングの法的問題」ホワイトペーパー（2023年）

## 戦略評価（strategy-advisor準拠）

### 開発優先度
- **総合判定: 中（長期展開推奨）**
- **理由**: 市場ニーズは確かに存在するが、スクレイピング法的リスクが初期段階での導入障壁となる。短期的には「手動入力ベース」での実装に限定し、スクレイピング技術は法的調査完了後のフェーズ2以降に導入すべき。

### 想定ROI（初期投資回収期間）
- **初期投資**: 開発費（スクレイピングなし版）300万 + 営業・マーケティング初期費用150万 = 450万円
- **月額平均客単価**: 3万円（手動入力モデルの場合）
- **想定初期採用速度**: 6か月で15社、12か月で35社
- **初期12か月MRR**: 15社×3万円=45万円 → 35社×3万円=105万円
- **回収期間**: 初期12か月で約100万円×12か月 = 1,200万円 → 4年程度で回収
- **ROI指標**: 初期投資450万 / 4年LTV（35社×3万円×48か月）= 5,040万 → **ROI: 11.2倍（4年ベース）**

### 成功指標（KPI）
- **短期（6か月）**: ベータテスト企業8社からの「営業資料として実用度」70%以上の評価獲得
- **中期（12か月）**:
  - 有料顧客: 35社
  - MRR: 105万円
  - 顧客が生成した比較資料の「営業で実際に使用された」割合: 70%以上
  - NPS: 45以上
- **長期（24か月）**:
  - 有料顧客: 80社（スクレイピング版への切り替え含む）
  - MRR: 240万円
  - スクレイピング版のベータテスト開始 → 初期10社

### コンティンジェンシープラン（リスク対応）
**リスク1: スクレイピング法的問題が予想以上に複雑化する場合**
- 対応: 企業の公開API活用（有限会社API等）を優先。手動入力モデルでのビジネス継続。

**リスク2: 競合企業からの法的請求が発生した場合**
- 対応: 顧問弁護士との相談体制構築。情報の「要約表示」に限定して著作権侵害リスクを軽減。

**リスク3: LLMの比較分析精度が60%以下の場合**
- 対応: 人手による品質チェック機能を搭載。ユーザーフィードバック基に継続改善。
