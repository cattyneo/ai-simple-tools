# 営業日報 AI 要約・分析ツール

## 概要
営業担当の日報（テキスト入力）を AI が自動要約し、チーム全体の傾向（頻出課題・受注パターン・失注理由等）をダッシュボード化。営業マネージャーの日報チェック時間を削減し、施策立案を加速。

## 目的・解決する課題
- 営業マネージャーが膨大な日報チェックに時間を費やしている
- チーム全体の受注パターン・失注要因が埋もれており、経営判断に活かされていない
- 属人的な営業手法を組織の資産化できていない

## ターゲット
営業組織20名以上で、日報システム（Salesforce・kintone等）を使用している企業。営業マネージャーが複数名いるミッドサイズ企業向け。

## 主要機能
- 日報の自動要約（AI が重要ポイントを抽出）
- 顧客の「課題キーワード」を自動抽出・集計（「〇〇が課題」と何社で出現したか）
- 受注案件と失注案件の自動分類・比較分析
- チーム内の「ベストプラクティス営業」の行動パターンを自動検出
- 失注要因の自動分類（「予算がない」「他社を選定」「ニーズなし」等）
- ダッシュボード化：日報集計→グラフ・表での可視化
- 営業マネージャー向けの「本日の気付き」サマリーをメール自動配信

## 競合との差別化
日報管理システム（Salesforce、kintone等）は基本的な集計機能のみ。本ツールは「AI による自動解析」により、隠れた営業ノウハウやチーム課題を浮き彫りにする。営業分析 SaaS（Veeva 等）は医療業界特化で高額。

## 想定価格帯
初期費用: 20万円 / 月額: 2～3.5万円（ユーザー数に応じた従量課金）

## 開発難易度
低～中 / 日報テキストの NLP（固有表現認識、テキスト分類）+ 集計ロジック + ダッシュボード表示で実装可能。既存 SFA（Salesforce等）との API 連携でデータソース多様化が可能。

## Agent所見
- **ディレクター**: 営業マネージャーの業務時間削減が明確で、組織スケーラビリティが向上する。日報の品質（どれだけ詳細に書かれているか）が成果を左右するため、運用面でのサポートが重要。
- **エンジニア**: NLP による自動分類・集計がコアロジック。既存 SFA との API 連携を段階的に実装することで、ロックイン効果を狙える。統計的な「異常値検知」機能を追加すれば差別化になる。
- **マーケター**: 営業組織内の「隠れた課題を可視化」という訴求が刺さる。営業マネージャーのペイン（日報チェック時間）をダイレクトに解消する点を強調。
- **セールス**: 導入後の「運用サポート」が成功の鍵。日報データが蓄積された 3ヶ月目以降の分析レポートが営業マネージャーの評価につながり、リテンションが向上。固定費モデルで提案すべき。

## 市場リサーチ補足（market-research準拠）

### 市場規模推定
- **TAM（Total Addressable Market）**: 日報システム導入企業（日本国内約3,000～5,000社） × 月額2～3.5万円 ≈ 約72～210億円
- **SAM（Serviceable Addressable Market）**: 日報システム導入 + 営業組織20名以上 ≈ 約1,500～2,500社 ≈ 約36～105億円
- **SOM（Serviceable Obtainable Market）**: 初期3年で狙える市場 ≈ 150～300社 × 年単価36万円 ≈ 5.4～10.8億円

### 競合実態
- **Salesforce、kintone等の日報機能**: 基本的な集計のみ。AI自動解析なし
- **Veeva**: 営業分析SaaS。医療業界特化で高額（月数十万円）。汎用性なし
- **HubSpot分析機能**: リード・案件分析機能がメイン。日報AI解析は付随的
- **本ツールの差別化**: 「日報テキストのAI自動解析」「失注要因の自動分類」「ベストプラクティス営業の自動検出」の統合機能。廉価性で優位。

### 反証・リスク要因（このツールが売れない可能性）
- **日報品質のばらつき**: 日報が不十分にしか書かれていない企業では、AI解析の精度が大幅に低下する
- **営業組織の抵抗**: 日報内容の「監視」と感じられ、営業スタッフからの反発が大きい可能性
- **ダッシュボード活用の難しさ**: 営業マネージャーがダッシュボード情報を「具体的な営業施策」に変換できず、インサイトが活かされない
- **既存SFA機能との競合**: Salesforce等の既存SFAも日報分析機能を継続更新しており、本ツール不要化の可能性
- **日報データの鮮度**: リアルタイムではなく「1日遅延」になるため、営業マネージャーの即座の対応ニーズに対応できない

### データソース
- Salesforce、kintone公式ドキュメント（日報管理・分析機能）
- Veeva公式プライシングページ
- 営業組織の日報運用調査（営業組織研究所、2023年版）
- Gartner「営業分析ツール選定ガイド」（2024年版）

## 戦略評価（strategy-advisor準拠）

### 開発優先度
- **総合判定: 中**
- **理由**: 営業マネージャーのペインは明確だが、実装後の「運用サポート」負荷が高い。日報データの品質がバラバラであり、AI解析精度が顧客企業に大きく左右される。ただし導入後3か月以降の「分析インサイト」が営業成果に直結すれば、リテンション率が高くなる可能性あり。

### 想定ROI（初期投資回収期間）
- **初期投資**: 開発費300万 + 営業・マーケティング初期費用100万 = 400万円
- **月額平均客単価**: 2.5万円（年間30万円）
- **想定初期採用速度**: 6か月で12社、12か月で35社
- **初期12か月MRR**: 12社×2.5万円=30万円 → 35社×2.5万円=87.5万円
- **回収期間**: 初期12か月で約65万円×12か月 = 780万円 → 6か月程度で回収
- **ROI指標**: 初期投資400万 / 3年LTV（35社×2.5万円×36か月）= 3,150万 → **ROI: 7.9倍（3年ベース）**

### 成功指標（KPI）
- **短期（6か月）**: ベータテスト企業8社からの「分析ダッシュボードの有用性」70%以上の評価獲得
- **中期（12か月）**:
  - 有料顧客: 35社
  - MRR: 87.5万円
  - ユーザーNPS: 55以上
  - 「ダッシュボード情報が営業施策に活かされた」という顧客の実感: 60%以上
- **長期（24か月）**:
  - 有料顧客: 80社
  - MRR: 200万円
  - 顧客の「営業チーム全体の受注率向上」: 平均15%以上改善
  - 顧客紹介による新規リード: 全新規リードの35%以上

### コンティンジェンシープラン（リスク対応）
**リスク1: 日報品質のばらつきにより、AI解析精度が低い場合**
- 対応: 日報作成ガイダンス機能を搭載。ユーザーフィードバック基に機械学習モデル継続改善。「日報品質スコア」を可視化して改善促進。

**リスク2: 営業スタッフからの「監視」反発が大きい場合**
- 対応: マネージャーアカウント限定の機能設計。スタッフ側には「個人のベストプラクティス検出」という前向きなメッセージで訴求。

**リスク3: Salesforce等が日報分析機能を充実化した場合**
- 対応: 「日報AI解析特化」「ベストプラクティス営業の自動検出」での差別化強調。既存SFAとの連携機能強化。
