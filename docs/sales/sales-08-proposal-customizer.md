# 提案書テンプレート自動カスタマイズツール

## 概要
業種×課題×企業規模に応じて、提案書テンプレート（PowerPoint 等）の文面・数値・事例を自動差し替え。営業が「刺さる提案書」を 5～10 分で作成できる。Mazrica Target の提案書特化版。

## 目的・解決する課題
- 提案書作成に営業が多くの時間を費やしており、商談スピードが遅い
- 提案書のクオリティが営業スキルに依存しており、受注率にばらつきがある
- テンプレートはあるが、毎回カスタマイズが必要で手間

## ターゲット
営業組織30名以上で、提案書による商談が多い業界（SaaS、コンサルティング、システムインテグレーター等）。営業マネージャーが「提案書の質を統一したい」と感じている層。

## 主要機能
- 業種（金融・製造・小売等）×課題（営業効率化・コスト削減等）の組み合わせで、提案書テンプレートを自動選択
- テンプレート内の「〇〇社は年 XX 万円のコスト削減」といった数値を、顧客企業規模に応じて自動調整
- 成功事例を顧客業種に合致したものに自動差し替え
- PowerPoint、Word、PDF フォーマットでの一括出力対応
- ブランドカラー・ロゴの自動適用
- 顧客名・商談日等の基本情報は CRM から自動取得
- 複数パターン提案書の自動生成（A 案・B 案・C 案）

## 競合との差別化
Mazrica Target は「提案書自動生成」をうたうが、実際はアウトラインのみで、営業が文面を調整する必要がある。本ツールは「テンプレート差し替え + 数値調整」により、完成度の高い提案書を即座に作成可能。Salesforce の提案管理ツール等は高額で複雑。

## 想定価格帯
初期費用: 45万円 / 月額: 3～5万円（提案書作成数に応じた従量課金）

## 開発難易度
中 / PowerPoint API（python-pptx 等）による自動生成 + テンプレート管理データベース + LLM による数値・事例の自動差し替えロジックで実装可能。テンプレート設計（パラメータ化）がポイント。

## Agent所見
- **ディレクター**: 営業の提案書作成時間が 80% 削減でき、商談スピードが向上するため、受注加速の経営KPI に直結。テンプレート設計時の業界/課題セグメント分析が初期導入のカギ。
- **エンジニア**: PowerPoint 自動生成 + テンプレート管理システム + 簡易 NLP による数値差し替えで実装可能。CRM（Salesforce等）との API 連携を早期に実装すると、データソースの信頼性が上がる。
- **マーケター**: 「営業スキルに左右されない提案書品質の統一」という訴求が、営業マネージャーに刺さる。ビフォーアフター動画（提案書作成の時短風景）による視覚的訴求が効果的。
- **セールス**: Mazrica Target ユーザーやセールスイネーブルメント機能を求めている企業がターゲット。導入時に「3 つの業種 × 3 つの課題 = 9 パターン」のテンプレート初期設定を無料サービスすることで、導入ハードルを低減。

## 市場リサーチ補足（market-research準拠）

### 市場規模推定
- **TAM（Total Addressable Market）**: 提案書作成プロセスがある企業（日本国内約6,000～10,000社） × 月額3～5万円 ≈ 約216～600億円
- **SAM（Serviceable Addressable Market）**: SFA導入企業で営業組織30名以上 ≈ 約1,500～2,500社 ≈ 約54～150億円
- **SOM（Serviceable Obtainable Market）**: 初期3年で狙える市場 ≈ 100～200社 × 年単価60万円 ≈ 6～12億円

### 競合実態
- **Mazrica Target**: 提案書自動生成を標榜。実際はアウトラインのみで、営業が文面調整が必須
- **Salesforce Proposal Generation**: ネイティブ機能。高額。汎用性が低い
- **Proposify**: 海外ツール。提案書テンプレート中心。自動カスタマイズ機能が弱い
- **本ツールの差別化**: 「テンプレート差し替え」「数値自動調整」「複数パターン自動生成」の統合機能。完成度の高い提案書をワンクリックで生成。

### 反証・リスク要因（このツールが売れない可能性）
- **テンプレート設計の複雑性**: 業界・課題による「提案書フォーマットの違い」が予想以上に大きく、テンプレート化不可能なケースが多い
- **数値自動調整ロジックのエラー**: 企業規模に応じた数値自動調整で「明らかに不正確な数値」が提示される可能性
- **提案書テンプレートのメンテナンス負荷**: 新しい業界・課題へのテンプレート追加時に、カスタマイズ工数が予想以上に増加
- **営業スキル依存の継続**: 実際には「提案書テンプレートの質」よりも「営業担当者の顧客理解度」が提案成功を左右するため、本ツールの導入効果が限定的
- **市場规模の縮小リスク**: デジタル化により「提案書を作る」という営業プロセス自体が短縮される可能性

## 戦略評価（strategy-advisor準拠）

### 開発優先度
- **総合判定: 中～低**
- **理由**: テンプレート設計が複雑で、初期導入時の構築工数が大きい。ROI回収期間が長い。ただし「提案書作成時間削減」というペインは実在するため、SFA導入企業をターゲットにすれば市場機会はあり。中期フェーズ（初期12～18か月後）での導入が適切。

### 想定ROI（初期投資回収期間）
- **初期投資**: 開発費600万 + 営業・マーケティング初期費用200万 = 800万円
- **月額平均客単価**: 4万円（年間48万円）
- **想定初期採用速度**: 6か月で10社、12か月で25社
- **初期12か月MRR**: 10社×4万円=40万円 → 25社×4万円=100万円
- **回収期間**: 初期12か月で約75万円×12か月 = 900万円 → 12か月程度で回収
- **ROI指標**: 初期投資800万 / 3年LTV（25社×4万円×36か月）= 3,600万 → **ROI: 4.5倍（3年ベース）**

### 成功指標（KPI）
- **短期（6か月）**: ベータテスト企業5社からの「提案書作成時間削減」50%以上の実績報告
- **中期（12か月）**:
  - 有料顧客: 25社
  - MRR: 100万円
  - 顧客の「提案書作成にかかる時間削減」: 平均60%以上改善
  - ユーザーNPS: 50以上
- **長期（24か月）**:
  - 有料顧客: 60社
  - MRR: 240万円
  - 本ツール利用による「提案承認率向上」: 平均10%以上改善
  - 顧客紹介による新規リード: 全新規リードの25%以上

### コンティンジェンシープラン（リスク対応）
**リスク1: テンプレート設計が複雑で、業界カバー率が低い場合**
- 対応: 初期段階では「SaaS業界」「コンサルティング業界」など2～3業種に限定。フェーズ2以降に追加業種対応。

**リスク2: 数値自動調整ロジックのエラー率が高い場合**
- 対応: ユーザーによる手動編集機能充実化。AI提案数値の「信頼度スコア」表示。

**リスク3: Salesforce等が提案書生成機能を充実化した場合**
- 対応: 「提案書特化」「複数パターン自動生成」での差別化強調。業界別テンプレートライブラリの充実。
