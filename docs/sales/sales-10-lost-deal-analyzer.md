# 失注分析・再アプローチツール

## 概要
過去の失注案件を AI が分析し、再アプローチのタイミング・切り口を自動提案。CRM に埋もれた失注データを資産に変換し、失注案件からの受注回収を実現。

## 目的・解決する課題
- 失注案件がそのまま CRM に放置されており、再営業の施策がない
- 失注理由（予算不足、タイミング不適切、競合選定等）が記録されていない、あるいは分析されていない
- 失注してから時間経過後の「今、再アプローチすべき案件」が不明確

## ターゲット
営業組織50名以上の BtoB 企業。営業サイクルが長い業界（システムインテグレーション、コンサルティング、SaaS 等）で、失注から数ヶ月後の再営業が効果的な業種。CRM（Salesforce等）に 1 年以上の失注履歴がある企業。

## 主要機能
- CRM の失注案件を自動抽出・分類（失注時期、失注理由を AI が推定）
- 失注理由別の分析：「予算がない」「他社を選定」「ニーズなし」等の分類
- 失注後の経過時間を考慮した「今、再アプローチすべき案件」を AI が提案
- 再アプローチ用のメール文面を自動生成（失注時の課題＋新機能＋成功事例を盛り込む）
- 失注企業の最新情報（人事異動・事業拡大等）を自動検知し、再営業のトリガーを提示
- 業界別・企業規模別の「失注からの受注確度」を可視化
- 失注案件の「生存期間分析」（失注後 6 ヶ月以内に再営業すると成功率 X%等）

## 競合との差別化
失注分析機能を持つ SFA（Salesforce等）は存在するが、「AI による自動提案」まで統合したツールは少ない。Datahug や Conversica は営業生産性ツールだが、失注分析に特化していない。CRM 単体では失注企業の最新情報取得・再アプローチ提案ができない。

## 想定価格帯
初期費用: 30万円 / 月額: 2.5～4万円（分析対象の失注案件数に応じた従量課金）

## 開発難易度
中 / CRM API での失注案件データ取得 + NLP による失注理由分類 + Web スクレイピングで企業情報更新検知 + LLM による再営業メール生成で実装可能。失注理由の推定精度が成果を左右するため、機械学習モデルの学習データが必須。

## Agent所見
- **ディレクター**: 既存顧客化を促進する「失注からの受注」というコンセプトが、営業効率の観点から極めて魅力的。ただし失注理由の正確な記録が前提となるため、導入前の CRM データ整備が重要。
- **エンジニア**: CRM API 連携 + NLP による失注理由分類がコア。Web スクレイピングで企業情報更新を検知する仕組みは、スケーラビリティを考慮した設計が必須。統計モデル（生存期間分析）の導入で科学的な根拠を提供。
- **マーケター**: 「失注は終わりではなく、資産である」という視点が営業マネージャーに刺さる。営業効率化ではなく「失注回収による売上向上」という切り口で訴求すると差別化できる。
- **セールス**: CRM 導入企業で、かつ営業サイクルが長い業界（SI、コンサル）がターゲット。初期導入時に「過去 1 年の失注案件の再営業予測レポート」を無料作成して、ツール効果を実感させることが受注のカギ。失注理由の分類精度を数値で提示（「85% の精度で失注理由を推定」等）することで信頼感を醸成。

## 市場リサーチ補足（market-research準拠）

### 市場規模推定
- **TAM（Total Addressable Market）**: 営業サイクルが長い業界の企業（日本国内約3,000～5,000社） × 月額2.5～4万円 ≈ 約90～240億円
- **SAM（Serviceable Addressable Market）**: SI・コンサルティング企業でCRM導入 + 営業組織30名以上 ≈ 約600～1,000社 ≈ 約18～40億円
- **SOM（Serviceable Obtainable Market）**: 初期3年で狙える市場 ≈ 80～150社 × 年単価48万円 ≈ 3.84～7.2億円

### 競合実態
- **Datahug**: 営業生産性ツール。失注分析機能は付随的
- **Conversica**: 営業AI支援。失注分析に特化していない
- **Salesforce分析機能**: 失注分析ダッシュボード存在。ただし「再営業提案」機能なし
- **本ツールの差別化**: 「失注分析 × AI再営業提案 × 企業情報更新検知」の統合機能。失注からの受注回収に特化。

### 反証・リスク要因（このツールが売れない可能性）
- **失注理由の記録品質**: CRM内に失注理由が十分に記録されていない企業が多く、分析基盤が不安定
- **Web スクレイピングの法的リスク**: 失注企業の最新情報取得時にスクレイピング利用の可能性があり、法的リスク
- **失注企業への再営業の心理的障壁**: 実は営業組織は「一度失注した企業への再営業」に抵抗を感じており、AI提案も活用されない可能性
- **機械学習モデルの精度確保**: 失注理由の分類精度が70%以下に留まる場合、顧客信頼度が大幅に低下
- **市場規模の限定性**: 営業サイクルが短い業界（EC、ネット広告等）ではニーズが限定的

### データソース
- Datahug、Conversica公式ドキュメント
- Salesforce失注分析機能公式ドキュメント
- CRM導入企業の失注管理実態調査（2023年版）
- 営業サイクル別の業界分析（IDC Japan）

## 戦略評価（strategy-advisor準拠）

### 開発優先度
- **総合判定: 低～中**
- **理由**: 営業サイクルが長い業界に限定され、市場規模が限定的。失注理由の記録品質がバラバラで、AI分析精度の確保が課題。ただし「失注からの受注」というコンセプトは魅力的で、初期採用企業からの推奨による新規顧客獲得期待は高い。長期フェーズ（初期18～24か月後）での投入が適切。

### 想定ROI（初期投資回収期間）
- **初期投資**: 開発費500万 + 営業・マーケティング初期費用150万 = 650万円
- **月額平均客単価**: 3.5万円（年間42万円）
- **想定初期採用速度**: 6か月で8社、12か月で20社
- **初期12か月MRR**: 8社×3.5万円=28万円 → 20社×3.5万円=70万円
- **回収期間**: 初期12か月で約55万円×12か月 = 660万円 → 12か月程度で回収
- **ROI指標**: 初期投資650万 / 3年LTV（20社×3.5万円×36か月）= 2,520万 → **ROI: 3.9倍（3年ベース）**

### 成功指標（KPI）
- **短期（6か月）**: ベータテスト企業5社からの「失注理由の分類精度」80%以上の評価獲得
- **中期（12か月）**:
  - 有料顧客: 20社
  - MRR: 70万円
  - ユーザーNPS: 48以上
  - AI提案に基づいて再営業を実施した顧客割合: 60%以上
- **長期（24か月）**:
  - 有料顧客: 50社
  - MRR: 175万円
  - 失注企業からの「再営業による受注」実績: 失注案件全体の5%以上
  - 顧客紹介による新規リード: 全新規リードの40%以上（失注回収による満足度向上）

### コンティンジェンシープラン（リスク対応）
**リスク1: 失注理由の記録品質が低く、AI分析精度が60%以下の場合**
- 対応: 導入前のCRMデータ整備サービス提供。失注理由の記録ガイドラインを顧客に提供。機械学習モデルの継続改善。

**リスク2: Web スクレイピング法的問題が発生した場合**
- 対応: 企業公開情報APIの活用（有限会社等のAPI）に切り替え。手動データ入力オプション提供。

**リスク3: 再営業への営業組織の抵抗が大きい場合**
- 対応: 「失注は機会」というポジティブな訴求強化。失注回収成功事例のケーススタディ化。経営層向けの「失注回収による売上向上」メリット説明資料提供。
