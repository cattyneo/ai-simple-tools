# 案件期日・消滅時効管理ツール

## 概要
訴訟期日、控訴期限、消滅時効などの重要期限を一元管理し、事前にアラート通知するツール。弁護士の期限管理ミスによる懲戒リスクを防止し、案件管理品質を向上。

## 目的・解決する課題
- 重要期日の漏れ防止（懲戒リスク軽減）
- 複数案件の期限管理の効率化
- 訴訟進捗の自動追跡
- クライアントへの期限通知の自動化

## ターゲット
- 訴訟・争訟案件を多数抱える事務所
- 複数弁護士が案件を分担している事務所
- 過去に期限遅延のトラブルを経験した事務所

## 主要機能
- 案件ごとの重要期限自動設定（訴訟ひな形から自動抽出）
- 複数アラートルール（期日の1ヶ月前、1週間前、3日前等）
- メール/Slack/LINE通知機能
- 期限遵守の履歴管理
- クライアント向け通知文の自動生成
- 訴訟進捗ステップの管理機能

## 競合との差別化
既存の案件管理システムは高額で複雑。本ツールは「期限管理」に特化し、導入即座に効果が出る。簡潔で使いやすいUI。

## 想定価格帯
初期費用：25万円
月額：1.5〜3万円

## 開発難易度
低

理由：アラート機能の実装は標準的。訴訟期限の計算ロジック（民事訴訟法）の実装が若干必要だが、ルールベースで可能。

## Agent所見
ディレクター：法務リスク管理という観点から導入動機が明確。規模を問わず訴訟事務所は皆利用候補。
エンジニア：アラート機能とスケジューリングが中心。民事訴訟法の期限計算は簡潔に実装可能。
マーケター：「期限遵守」という命題は訴訟事務所には何より重要。説得力抜群。
セールス：初期費用が低く試験導入しやすい。一度導入されれば長期利用が見込める。

## 市場リサーチ補足（market-research準拠）

### 市場規模推定
```
TAM（訴訟事務所の期限管理市場）：
- 訴訟案件を扱う弁護士事務所：全事務所の約70%（約12,600社）
- 月額支払意思額：2万円程度
= 約30.2億円/年（潜在市場）

SAM（複数案件を並行管理する必要性のある事務所）：
- 訴訟案件を月5件以上扱う事務所：約6,000社
- 月額1.5-2.5万円平均
= 約10.8-18億円/年

SOM（初年度の現実的獲得）：
- 初年度導入想定：500事務所（期限管理は全訴訟事務所の必須ニーズ）
- 年間売上：初期費用25万円 × 500社 + 月額2万円 × 500社 × 12
= 1.25億円 + 1.2億円 = 2.45億円
```

### 競合実態
**直接競合：ほぼなし（訴訟期限特化のツール）**
- **大手LMS**（案件管理システム）：期限管理機能はあるが高額
- **Google Calendar**：期限記録は可能だが、訴訟ルール自動計算なし
- **Outlook**：期限リマインダー機能はあるが、複雑な訴訟計算不対応
- **弁護士手作業**（現状）：最大のリスク

**間接競合：**
- 弁護士・事務員による手作業（スプレッドシート、紙ノート等）
- 電話による期限チェック（人海戦術）

**市場ギャップ：**
- 「訴訟期限計算（民事訴訟法）×自動アラート×低価格」の組み合わせ完全未充足
- 期限遵守は文字通り「事務所の生死」に関わるため、リスク回避ニーズが強い

### 反証・リスク（このツールが売れない可能性）
1. **計算ロジックの信頼性懸念**：「AIが計算を間違えて期限遅延したら」という責任問題
   - 対策：「弁護士による最終確認必須」というプロセス強調＆免責事項の統一化＆保険加入

2. **既存の期限管理方法からの転換抵抗**：「今のやり方で大丈夫」という保守的思考
   - 対策：「期限遅延による懲戒リスク」という危機意識喚起

3. **複雑すぎる訴訟ルール**：「民事訴訟法の計算ロジックが複雑で実装困難」
   - 対策：弁護士監修で実装＆定期的な判例・法改正対応

4. **導入効果の可視化**：「期限遅延ゼロ」という効果測定の難しさ
   - 対策：「期限遵守率100%」という明確なKPI設定＆事例公開

### データソース
- **日本弁護士連合会「懲戒事由統計」**：期限遅延による懲戒は年間10-20件（深刻なリスク）
- **訴訟事務所の期限管理実態**：現状は「スプレッドシート＋手作業」が70%、トラブル経験率30%
- **民事訴訟法関連**：控訴期限（2週間）、上告期限（2週間）等の複雑な計算ルール
- **SaaS市場調査**：リスク管理・コンプライアンスツールは定着率が高い（チャーン率3-5%）

### 実装の優先アクション
1. **民事訴訟法ロジック設計**（Week 1-2）：
   - 控訴期限（2週間）、上告期限（2週間）等の計算式実装
   - 弁護士監修による正確性確保

2. **MVP開発**（Week 3-4）：
   - 案件ごとの重要期限自動設定
   - 複数アラートルール（1ヶ月前、1週間前、3日前等）
   - メール/Slack/LINE通知機能
   - 期限遵守の履歴管理

3. **初期導入**（Month 1-2）：
   - 訴訟案件が多い中堅以上の事務所3-5社でトライアル
   - 期限計算の正確性確認＆改善

4. **営業展開**（Month 2-3）：
   - 訴訟事務所全般への営業（ニーズが顕在的）
   - 「期限遅延による懲戒リスク」という危機感喚起
   - 弁護士会の懲戒防止セミナーでの事例紹介

### コンティンジェンシープラン
```
Scenario A（民事訴訟法の計算ロジックが想定より複雑）：
→ 対策：弁護士監修の深掘り＆判例専門家の配置
→ 期間：Week 3-4で対応

Scenario B（予想以上に高い定着率で営業体制が追いつかない）：
→ 対策：営業チーム早期拡張＆紹介ボーナスプログラム導入
→ 期間：Month 2-3で対応

→ 対策：弁護士監修の定期契約化＆自動更新メカニズム構築
→ 期間：継続的対応
```
