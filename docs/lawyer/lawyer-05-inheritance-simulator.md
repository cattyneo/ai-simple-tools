# 相続計算シミュレーター（集客用）

## 概要
HPに埋め込み型の相続税・遺産分割シミュレーターを設置。潜在顧客が自分の家庭の相続について計算・試算しながら理解を深め、その後「詳しく相談したい」と相談予約に誘導するリードジェネレーションツール。

## 目的・解決する課題
- 相続に不安を持つ潜在顧客の早期キャッチ
- HPの滞在時間延長・離脱率低下
- 「相続」というニッチキーワードでSEO流入増加
- 相談予約前のクライアント学習投資（相談時間削減）

## ターゲット
- 相続案件を主力とする事務所
- 東京都内などの高資産家が多い地域の事務所
- Google検索で「相続 弁護士」を狙う事務所

## 主要機能
- 相続税自動計算（配偶者控除、基礎控除、特例等対応）
- 遺産分割パターンのシミュレーション
- 節税対策の提案（生前贈与など）
- 結果PDF出力＆メール配信機能
- シミュレーター結果からの相談予約ボタン
- 利用者の相続パターンの自動分析→マーケティングデータ化

## 競合との差別化
既存の相続税計算ツール（国税庁のweb等）は複雑で使いにくい。本ツールは弁護士事務所の「相談に繋ぐ」という意図で設計し、利用体験がスムーズ。節税提案も含むため、より付加価値が高い。

## 想定価格帯
初期費用：60万円
月額：2〜4万円

## 開発難易度
中

理由：相続税計算ロジックの実装に若干の複雑性。法令改正への追従が定期的に必要（2025年相続税改正等）。シミュレーション結果の予約連携、PDFレポート生成が主な工数。

## Agent所見
ディレクター：相続市場は確実で大型。相談単価が高いため、見込み客一人の価値が大きい。
エンジニア：税務計算ロジックの実装が主課題。税理士監修が必須。
マーケター：「自分の相続税はいくら？」というUGCがSNS拡散する可能性。バイラル効果期待。
セールス：相続は「後悔しない決断」を求める層。説得力が高く受注率が良い。

## 市場リサーチ補足（market-research準拠）

### 市場規模推定
```
TAM（相続関連サービス市場）：
- 相続件数/年：約150万件（日本の高齢化で増加中）
- 相続税申告件数/年：約12万件
- 相続関連サービスの平均単価：100万円以上
= 約150億円超（潜在市場）

SAM（相続に力を入れている小〜中堅弁護士事務所セグメント）：
- 相続案件を主力とする事務所：約3,000社
- 月額シミュレーター導入意思：2-5万円
= 約7.2-18億円/年

SOM（初年度の現実的獲得）：
- 初年度導入想定：200事務所（相続ニッチは営業成功率高い）
- 年間売上：初期費用60万円 × 200社 + 月額3万円 × 200社 × 12
= 1.2億円 + 0.72億円 = 1.92億円
```

### 競合実態
**直接競合：存在する（税務計算ツール等）**
- **国税庁の相続税計算シミュレーター**：完全無料だが、UI複雑で一般向けに不親切
- **相続税申告ソフト**（TKC, 勤務弁護士向け）：税理士向けで弁護士事務所向けではない
- **相続税自動計算サイト**（複数あり）：汎用だが法律相談への誘導機能なし
- **税理士のシミュレーション機能**：高額で弁護士事務所の顧客ニーズに対応していない

**競合の弱点：**
- UI/UXが複雑すぎて一般ユーザーが理解困難
- 相談への誘導フローがない（単独ツールで終わる）

**市場ギャップ：**
- 「一般向けUI＋弁護士への自然な相談誘導」の組み合わせ不充足

### 反証・リスク（このツールが売れない可能性）
1. **税務計算の正確性懸念**：「間違った計算で顧客を誤導するリスク」
   - 対策：税理士監修の明記＋「参考値」表示＆「必ず税理士・弁護士に相談」の警告

2. **税理士との競合関係**：「税務計算は税理士の領域」という反発
   - 対策：「相続税+遺産分割法律相談の前段階」というポジショニング

3. **相続案件の地域偏差**：「都市部（高資産家）と地方で利用価値が異なる」
   - 対策：地域別の資産価格データベース搭載＆カスタマイズ

4. **マーケティング学習コスト**：「弁護士事務所がシミュレーターをどう活用するか」の知識不足
   - 対策：「相続シミュレーター マーケティング活用ガイド」の無料提供

### データソース
- **相続税改正情報**（国税庁、2025年改正予定）：基礎控除引き下げで課税対象者増加
- **相続案件の市場規模**（矢野経済研究所「相続関連市場調査」）：年間12万件の相続税申告、平均相談単価100-200万円
- **弁護士事務所ヒアリング**：相続案件の営業リード数が不足＆相談予約率が20-30%（他案件の50-70%より低い）という実態
- **シミュレーターの活用効果**（ネット生保等の実績）：シミュレーター利用者の契約率は非利用者の1.5-2倍

## 戦略評価（strategy-advisor準拠）

### 開発優先度
**中〜高**

**根拠：**
- 開発難度：中（相続税計算ロジック＋遺産分割シミュレーション）
- 市場規模：大（相続市場は確実で高単価）
- 導入障壁：中程度（相続ニッチに限定＆営業ターゲットが明確）
- 継続性：中（毎年の法改正対応が必要＆利用頻度は相対的に低い）
- 営業効率：高い（相続案件に力を入れている事務所への営業は成功率高い）
- リード品質：最高（相談予約への誘導が強い＆相談単価が高い）

**優先順位：** TOP3から外れるがフェーズ2（Month 4-9）での開発推奨

### 想定ROI
```
初期投資：1,200万円（開発+税理士監修+法改正対応体制）
初期費用：60万円（顧客単価：高い）
月額費用：3万円（平均）

1事務所の3年間LTV：
= 60万円（初期） + 3万円 × 36ヶ月
= 168万円

CAC：初期費用の50% = 30万円（営業難度中程度）
LTV/CAC比：5.6倍（良好）

初期投資回収期間：
1,200万円 ÷ (60万円初期 + 3万円×12月)
= 約1.7年（顧客200社突破時点）
```

### 成功指標（具体的KPI）
```
短期（6ヶ月）：
- 導入事務所数：60社（相続特化の営業難度）
- MRR：180万円
- シミュレーター利用者数：月1,000利用者超
- 相談予約への誘導率：30%以上

中期（12ヶ月）：
- 導入事務所数：200社
- MRR：600万円
- 「シミュレーター利用者の相談予約率」：目標30-40%
- 相続案件の受注件数増加率：導入事務所で+20%
- 法改正対応：年1回以上の自動更新完了

長期（24ヶ月）：
- 導入事務所数：400社
- ARR：1.92億円
- NPSスコア：50以上（ニッチながら満足度高い）
- 相続市場での認知度：「相続+弁護士」検索時の上位表示
```

### 実装の優先アクション
1. **税務計算ロジック設計**（Month 1-2）：
   - 税理士による監修＆改正相続税への対応
   - 配偶者控除、基礎控除等の複雑な計算式の実装

2. **MVP開発**（Month 2-3）：
   - 相続税自動計算機能
   - 遺産分割パターンのシミュレーション（複数相続人対応）
   - 節税対策の提案（生前贈与等）
   - PDF出力＆メール配信機能

3. **初期導入**（Month 3-4）：
   - 相続案件に力を入れている中堅事務所3-5社をターゲット
   - トライアル期間中の効果測定（相談予約率向上）

4. **営業展開**（Month 4-6）：
   - 相続に特化した弁護士事務所への営業
   - 「相続税改正2025年対応」というタイミング活用
   - 相続ナビサイト等への広告掲載

### コンティンジェンシープラン
```
Scenario A（税務計算の複雑性が想定以上）：
→ 対策：税理士監修のアウトソース拡大＆AIによる計算補助
→ 期間：Month 2-3で対応

Scenario B（相続税改正への追従コストが高い）：
→ 対策：自動更新メカニズムの開発＆税理士監修の定期契約化
→ 期間：Year 1から組織化

Scenario C（大手税務ソフト企業が弁護士向けに参入）：
→ 対策：「相続+遺産分割+紛争対応」の総合パッケージ化で差別化
→ 期間：Year 2以降の機能拡張
```
