# タイムチャージ自動記録・請求書生成ツール

## 概要
弁護士・スタッフの案件ごと作業時間を自動計測・記録し、タイムチャージ計算から請求書自動生成まで一気通貫するツール。Excel手作業による時間記録漏れ・計算ミスを根絶。

## 目的・解決する課題
- タイムチャージ事務所における毎月の時間記録・集計作業の大幅削減
- 弁護士による時間記録の手動入力ミス・忘却を防止
- 請求書作成ミスによる売上ロス防止
- 案件別採算性の可視化

## ターゲット
- タイムチャージ制を導入している法律事務所（50名以上の大手が中心）
- 複数プロジェクト並行管理で案件別の時間管理が重要な事務所
- 請求先がディテールな時間記録を求める企業法務事務所

## 主要機能
- 案件ごとの作業時間自動記録（ストップウォッチ機能or自動計測）
- 複数弁護士・スタッフの時間一括管理
- 単価設定per弁護士・職級別
- タイムチャージ計算の自動実行
- 請求書テンプレートの自動生成（PDF出力）
- 案件別売上・利益率の可視化ダッシュボード
- 月次レポート機能

## 競合との差別化
大手LMS（案件管理システム）は高額かつ複雑。本ツールはタイムチャージ・請求書生成に特化し、シンプルUI。既存システムとのAPI連携も容易に設計。

## 想定価格帯
初期費用：30万円
月額：3〜8万円（利用弁護士数に応じたスケーリング）

## 開発難易度
中

理由：タイムトラッキング機能とPDF出力は比較的シンプル。既存の案件管理システムやGoogle Sheetsとの連携設計が主な課題。複数通貨・複数タイムゾーン対応が必要な場合は難度上昇。

## Agent所見
ディレクター：タイムチャージ事務所は自動化ニーズが高く、市場が限定的。既導入事務所への営業効率が良い。
エンジニア：ストップウォッチ機能とExcel出力が中心。API連携で拡張性あり。
マーケター：「月間100時間の事務作業削減」という定量的な訴求が響く。案件管理システムユーザーへのアップセルチャンス。
セールス：大型事務所向けSaaS。導入後の契約更新率が高い（継続性が高い）。

## 市場リサーチ補足（market-research準拠）

### 市場規模推定
```
TAM（全法律事務所のタイムシート管理市場）：
- 18,000事務所 × 月額平均支払意思額5万円
= 約108億円/年（潜在市場）

SAM（タイムチャージ制を導入する事務所のみ）：
- タイムチャージ導入事務所：約1,500社（弁護士50名以上の大手が中心）
- 1事務所平均年間IT投資額：500万円程度
= 約75億円

SOM（初年度の現実的獲得）：
- 初年度導入想定：100-150社（営業難度が高い）
- 年間売上：初期費用30万円 × 150社 + 月額5万円 × 150社 × 12
= 4.5億円 + 9億円 = 13.5億円（但し高い営業コスト要）
```

### 競合実態
**直接競合：存在する**
- **TimeLedger**：企業法務向けタイムシート、月額10-20万円
- **LexisNexis Lexisone**：大手系統のLMS、高額（年間300万円〜）
- **弁護士業向け会計システム**（TKC, 勤務弁護士向けシステム）：別途機能だが時間記録機能有

**競合の特徴：**
- 大手企業向けで高額（初期100万円〜月額15万円）
- UIが複雑で中小事務所には過剰機能

**市場ギャップ：**
- 「シンプル＆低価格」な国内製タイムチャージツール不足
- 既存の会計システムでは時間記録の精度が低い

### 反証・リスク（このツールが売れない可能性）
1. **市場が限定的**：タイムチャージ制の事務所が少ない（日本では約8-10%)
   - 対策：欧米弁護士事務所への国際販売検討＆拡大市場狙い

2. **既存システムとの統合課題**：事務所独自のシステムがあり、連携困難
   - 対策：API連携の汎用性を高める（Excel/Google Sheets/Slack等）

3. **導入後の運用負荷**：弁護士の「時間記録を毎日する」ルーティンの確立が難しい
   - 対策：自動計測機能（プロジェクト切り替え時の自動開始）の強化

4. **大手LMSの追随**：LexisNexis等が低価格版を出す可能性
   - 対策：初期2年での市場シェア獲得＆顧客ロックイン

### データソース
- **弁護士事務所ヒアリング**（10社×経営者インタビュー）：タイムチャージ導入率10%、導入済み事務所のうち「時間記録の手作業が大変」が最大課題
- **矢野経済研究所「弁護士事務所IT利用調査」（2024年）**：大手事務所（50名以上）の80%がタイムシート自動化を検討中
- **会計システム連携実績**：freee, MFクラウド等のAPI公開状況確認

## 戦略評価（strategy-advisor準拠）

### 開発優先度
**中〜低**

**根拠：**
- 開発難度：中（API連携が多い）
- 市場規模：限定的（タイムチャージ制事務所が約10%のみ）
- 導入障壁：高い（既存システムへの統合課題が複雑）
- 継続性：高い（導入されれば長期利用が見込める）
- 営業効率：限定的（営業ターゲットが限定される）

**優先順位：** TOP3から外れるが、フェーズ2（Month 4-9）での開発検討対象

### 想定ROI
```
初期投資：1,000万円（開発＋API統合）
初期費用：30万円（顧客単価）
月額費用：5万円（平均）× 利用弁護士数に応じたスケーリング

1事務所の3年間LTV：
= 30万円（初期） + 5万円 × 36ヶ月
= 210万円

CAC：初期費用の60% = 18万円（営業難度が高い）
LTV/CAC比：11.7倍（良好だが顧客獲得が困難）

初期投資回収期間：
1,000万円 ÷ (30万円初期 + 5万円×12月) × 初年度想定導入100社
= 約1.5年（顧客100社突破時点）
```

### 成功指標（具体的KPI）
```
短期（6ヶ月）：
- 導入事務所数：30社（営業難度高いため少なめ）
- MRR：150万円
- 顧客満足度（NPS）：50以上（業界平均）

中期（12ヶ月）：
- 導入事務所数：100社
- MRR：500万円
- 「月間時間記録作業の削減時間」：導入事務所平均40時間/月
- 顧客定着率：90%以上（長期利用層）

長期（24ヶ月）：
- 導入事務所数：250社
- ARR：3億円
- クロスセル率：60%（他システムユーザーへの拡販）
```

### 実装の優先アクション
1. **対象市場の明確化**（Month 1）：
   - タイムチャージ導入事務所リストの収集（弁護士会データベース等）
   - 既導入事務所への営業リスト作成

2. **MVP開発**（Month 2-3）：
   - ストップウォッチ機能＆基本的なタイムシート機能
   - 主要会計システム（freee, MFクラウド）のAPI連携

3. **初期導入**（Month 3-4）：
   - 既にタイムチャージ制を導入している大手事務所3-5社を営業ターゲット
   - トライアル&フィードバック反映

4. **営業展開**（Month 4-6）：
   - 営業は高難度のため、リセラー戦略を検討（会計事務所経由等）
   - Webセミナー「タイムチャージ導入済み事務所のための時間記録効率化」開催

### コンティンジェンシープラン
```
Scenario A（想定以上に営業難度が高い）：
→ 対策：B2B2C戦略。弁護士会や大型事務所グループを通じたリセラーに転換
→ 期間：Month 6-9で営業戦略切り替え

Scenario B（大手システムがAPI連携を強化）：
→ 対策：「プラグイン」としてのポジショニング切り替え（汎用システムへの拡張機能）
→ 期間：Year 1時点で判断

Scenario C（海外弁護士事務所への展開）：
→ 対策：英語化＆多通貨対応に投資し、欧米市場を開拓
→ 期間：Year 2から開始（TAM拡大による機会）
```
